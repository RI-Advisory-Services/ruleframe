# Computed Columns (v1)

Computed columns are generated before rule evaluation and then treated like normal input columns.

## Example

```yaml
computed_columns:
  - id: total_measure_savings
    name: Total Measure Savings
    type: sum
    columns:
      - Total Measure Gross kWh
      - Total Measure Gross kW
      - Total Measure Gross Therms
```

## Supported Types

### Row-level arithmetic

- **`sum`** — add all values across a `columns` list (skips NaN with `min_count=1`)
- **`subtract`** — left-to-right subtraction across a `columns` list
- **`multiply`** — left-to-right product across a `columns` list
- **`divide`** — divide exactly 2 columns; NaN when denominator is zero

### Row-level logic

- **`coalesce`** — first non-null value per row across a `columns` list
- **`all_blank_or_zero`** — `1` if every column in `columns` is blank or zero, else `0`

### Date

- **`date_diff`** — `(end_column - start_column)` in whole days; requires `start_column` and `end_column`
- **`days_since_today`** — `(today - column)` in whole days; requires `column`
- **`years_since_year`** — `(current_year - column)` as an integer year count; requires `column`

### Group aggregates

- **`group_sum`** — sum a `value_column` grouped by `group_by`; optional `filter` restricts which rows contribute; non-contributing rows receive NaN
- **`group_count`** — count rows grouped by `group_by`; optional `filter`; non-contributing rows receive NaN

#### group_sum / group_count filter

```yaml
- id: project_bef_kwh
  name: Project BEF Total kWh
  type: group_sum
  group_by: Project ID
  value_column: kWh
  filter:
    column: Measure Type
    equals: BEF
```

Rows that do not match the filter receive NaN (not zero). This is intentional — a NaN means the value is undefined for that row, not zero.

## Chaining Computed Columns

A computed column may reference a previously generated computed column as one of its inputs. This supports patterns like:

```yaml
computed_columns:
  # Step 1: sum component savings
  - id: total_savings
    name: Total Savings
    type: sum
    columns:
      - kWh Savings
      - Therm Savings

  # Step 2: divide a column by the computed total (references step 1)
  - id: kwh_share
    name: kWh Share
    type: divide
    columns:
      - kWh Savings
      - Total Savings       # generated by step 1
```

**Order matters.** Computed columns are applied in declaration order. A column that depends on another generated column must be declared after it. There is no automatic sorting — the author is responsible for dependency order.

## Pipeline Position

1. Validate required source columns against the input DataFrame.
2. Apply computed columns in declaration order.
3. Evaluate rules using input columns plus all computed columns.

## Known Constraints

- **Dependency order** — computed columns that chain from other computed columns must appear after their dependency in the list. The package does not auto-sort. An out-of-order reference raises `BundleValidationError` at validation time.
- **Cycle detection** — cycles (A → B → A) raise `BundleValidationError` at validation time.
- **Self-reference** — a spec that lists its own output name as an input raises `BundleValidationError` at validation time.
- **Name collision with input columns** — if a generated column name matches an existing input column, `InputSchemaError` is raised before any data is processed. Avoid reusing input column names for computed columns.

1. Validate required source columns.
2. Compute generated columns.
3. Evaluate rules using input plus computed columns.
